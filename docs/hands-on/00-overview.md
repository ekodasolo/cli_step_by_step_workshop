# ハンズオン概要

## このワークショップでやること

AWS CLI を使って、以下のインフラ構成を **1 リソースずつ手で作ります**。

```
            Internet
               │
         ┌─────┴─────┐
         │    IGW     │
         └─────┬─────┘
               │
    ┌──────────┴──────────┐
    │        VPC          │
    │   (10.0.0.0/16)     │
    │  ┌───────┬────────┐ │
    │  │Pub-1a │ Pub-1c │ │
    │  │.1.0/24│.2.0/24 │ │
    │  │       │        │ │
    │  │  EC2  │  EC2   │ │  ← Auto Scaling Group
    │  └───┬───┴───┬────┘ │
    │      └───┬───┘      │
    │      ┌───┴───┐      │
    │      │  ALB  │      │
    │      └───────┘      │
    └─────────────────────┘
```

最終的に ALB の DNS 名にブラウザでアクセスすると、カレンダーページが表示されます。
リロードするたびに Instance ID と Private IP が変わり、ロードバランシングされていることを確認できます。

---

## 実行環境

- **AWS CloudShell** を使用します
- マネジメントコンソールにログインし、画面上部のターミナルアイコンから起動してください
- リージョンは **東京（ap-northeast-1）** を選択してください

## 手順書の読み方

各手順書は以下の構成になっています:

| セクション | 内容 |
|-----------|------|
| **About** | 何を作るかの概要 |
| **When — Before** | 作業の事前条件（何が準備済みである必要があるか） |
| **When — After** | 完了条件（何ができていれば OK か） |
| **How — 1. 前処理** | パラメータ準備、事前条件の確認 |
| **How — 2. 主処理** | リソースの作成（メインの作業） |
| **How — 3. 後処理** | 完了条件の確認 |
| **Navigation** | 次の手順へのリンク |

コマンドはすべて **コピー＆ペーストで実行** できます。
`output` ブロックに出力例を載せているので、自分の実行結果と見比べてください。

## 作成するリソースの全体像

手順が進むにつれて、以下のリソースが積み上がっていきます。

```
Step 01: VPC
Step 02:  └── Subnet 1a, Subnet 1c
Step 03:  └── IGW → Route Table → Route → Subnet 関連付け
Step 04:  └── ALB SG, EC2 SG
Step 05: IAM Role → Instance Profile
Step 06: EC2 (単体で動作確認)
Step 07: ALB → Target Group → Listener (EC2 を登録)
Step 08: Launch Template → ASG (EC2 を ASG に置き換え)
Step 09: ↑ 全部を逆順で削除
```

## 困ったときは

- コマンドの詳細は `aws <service> <command> help` で確認できます（例: `aws ec2 create-vpc help`）
- 出力例と異なる結果が出た場合は、変数の値が正しくセットされているか `echo ${変数名}` で確認してください
- エラーメッセージが出た場合は、メッセージ内容を講師に共有してください
